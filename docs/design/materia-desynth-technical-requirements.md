# 精選/分解 技術要件まとめ

本ドキュメントは、精選(マテリア精製)と分解に関する技術的な要件と現状仕様を整理したものです。
調査・設計・実装の前提として参照します。

## 1. 対象機能と目的
- 精選(マテリア精製): スピリットボンド100%の装備品を検出し、精選UIを自動操作する。
- 分解: 所持品のみを対象に、レベル範囲とモードに従って自動分解する。
- いずれもUIから状態/対象/実行を確認できること。

## 2. 共通要件
- プレイヤーログイン状態でのみ動作する。
- UI側で状態、対象件数、実行可否が分かること。
- 設定はConfigurationに保存し、再起動で復元されること。
- エラーや失敗はログに記録し、処理が止まりにくい設計とする。

## 3. 精選(マテリア精製)の要件

### 3.1 対象条件
- 対象範囲: 所持品 + アーマリーチェスト + 装備中。
- 判定: Spiritbond >= 10000 かつ MateriaSlotCount > 0。
- 取得元: InventoryService.GetExtractableItems()。

### 3.2 実行方式
- UI操作による自動化のみ。
- Materialize / MaterializeDialog のアドオンが表示中であることが前提。
- アドオン未表示時は処理を待機し、警告ログを出す。

### 3.3 UI要件
- ホームタブに「マテリア精製」セクションを配置。
- 状態/対象件数を表示。
- 「自動精製を有効化」チェックでON/OFF。

### 3.4 状態管理
- MateriaExtractService により状態遷移(Disabled/Scanning/Extracting/Waiting/Idle)。
- 有効化/無効化はConfiguration.MateriaExtractEnabledで保持。

### 3.5 制限/注意点
- Materializeウィンドウの自動オープンは未実装。
- Combat/Occupied状態中は基本的に操作を避ける。
  - ただし精選ウィンドウが開いている場合はOccupiedを許容する。

## 4. 分解の要件

### 4.1 対象条件
- 対象範囲: 所持品のみ(装備中/アーマリーチェストは対象外)。
- 判定: Item.Desynth > 0 かつ ItemLevelが範囲内。
- 取得元: InventoryService.GetDesynthableItems(min, max)。

### 4.2 UI要件
- ホームタブに「アイテム分解」セクションを配置。
- 状態/対象件数/レベル範囲/モード/スコープを表示。
- 分解開始は確認ダイアログを必須。

### 4.3 確認ダイアログ
- 対象件数/レベル範囲/スコープ/上位10件/高IL強調を表示。
- 0件の場合は開始ボタンを無効化。

### 4.4 状態管理
- DesynthService により処理状態を管理。
- 「分解停止」ボタンで停止可能。

## 5. 依存サービス/コンポーネント
- InventoryService: インベントリ/アーマリー/装備の走査と判定。
- InventoryCacheService: UI表示用のスナップショットキャッシュ。
- GameUiService: Materialize/SalvageなどUI操作。
- Configuration: 機能ON/OFFと各種設定。

## 6. 既知の制約と調査ポイント
- Materializeウィンドウ無しでの精選実行は現状不可。
  - UIを介さない方法が必要だが、Dalamud環境での可否は未調査。
- 精選対象が見つからない場合の原因調査:
  - 対象コンテナの範囲
  - Spiritbond判定
  - UIオープン状態
  - ConditionFlagの影響

## 7. 今後の検討項目
- 精選ウィンドウの自動オープン可否。
- UIを介さない精選実行の可能性調査。
- パッチ変更によるアドオン名/コールバック値の変化追従。
