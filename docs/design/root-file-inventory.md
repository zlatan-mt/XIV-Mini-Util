# ルート直下ファイル棚卸し

## 目的

この文書は、リポジトリルート直下のファイルを役割ごとに棚卸しし、今後の整理判断の基準を残すためのものである。  
対象は 2026-04-20 時点で `C:\Project\apps\XIV-Mini-Util` のルートに存在するファイルとする。

## 判断区分

- 残置
  - ルート直下に置く意味が明確で、入口ファイルとして維持する
- 条件付き残置
  - 直ちに動かさないが、運用次第で移動または削除を検討する
- 移動候補
  - ルートより別の配置先のほうが役割に合っている
- 削除候補
  - リポジトリ管理対象としての必要性が低い

## 棚卸し結果

### 残置

#### `README.md`

- 役割:
  - リポジトリの入口
- 理由:
  - 利用者と開発者の双方が最初に参照するため
- 判断:
  - ルート残置

#### `AGENTS.md`

- 役割:
  - リポジトリ固有の作業指示
- 理由:
  - エージェント系ツールがルートで検出しやすい位置にある必要がある
- 判断:
  - ルート残置

#### `CHANGELOG.md`

- 役割:
  - 変更履歴の入口
- 理由:
  - release や履歴確認時にルートにある価値が高い
- 判断:
  - ルート残置

#### `LICENSE`

- 役割:
  - ライセンス提示
- 理由:
  - 一般的にルート配置が標準
- 判断:
  - ルート残置

#### `pluginmaster.json`

- 役割:
  - Dalamud 配布導線の入口
- 理由:
  - 公開 URL として直接参照されるため、ルート配置が自然
- 判断:
  - ルート残置

#### `.gitignore`

- 役割:
  - Git 管理対象の制御
- 理由:
  - ルート配置が必須
- 判断:
  - ルート残置

### 条件付き残置

#### `CLAUDE.md`

- 役割:
  - Claude 系ツール向けの補助文書
- 理由:
  - 運用上参照される可能性がある
- 懸念:
  - `.gitignore` では AI ツール補助ファイルを除外する方針があり、公開リポジトリ上では必須性が低い
- 判断:
  - 当面は残置
  - 今後「ローカル専用」と定義するなら削除または未追跡化を検討

#### `.skillportrc`

- 役割:
  - ツール設定ファイル
- 理由:
  - ルートにあることでツールが検出しやすい可能性がある
- 懸念:
  - プロダクト本体の理解には不要
- 判断:
  - 当面は残置
  - 実運用で不要と判断できたら削除またはローカル限定へ移行

### 削除候補

#### `XivMiniUtil.zip`

- 役割:
  - ビルド済み配布物
- 理由:
  - GitHub Release asset として配布するものであり、通常はリポジトリ直下に常駐させる必要がない
- 懸念:
  - `.gitignore` でも `*.zip` は除外対象
  - バイナリをルートに置くと差分確認や整理性を悪化させる
- 判断:
  - リポジトリ管理対象から外す候補
  - 必要なら `artifacts/` や release 作業時の一時出力へ寄せる
